Classes, Objects & Features

OOP

A Class is a blueprint/template that defines attributes (fields) and behaviors (methods). An Object is a real-world instance of a class with its own state and identity.

  • Class = blueprint; Object = instance of that blueprint
  • Java is purely object-oriented (everything is inside a class)
  • Key OOP features: Encapsulation, Inheritance, Polymorphism, Abstraction
  • Objects are created using the new keyword
class Car { String brand; // field void drive() {} // method } Car c = new Car(); // object
⭐ Exam Tip: Define class vs object with a real-world analogy (e.g., "Dog" class → "Tommy" object).
Easy
🏗️

Object Declaration & Initialization

OOP

Declaration creates a reference variable. Initialization allocates memory and links the reference to an actual object using new.

  • Declaration: ClassName ref; — only a reference, no object yet
  • Initialization: ref = new ClassName(); — memory allocated on heap
  • Both can be combined: ClassName ref = new ClassName();
  • Uninitialized reference holds null
Student s; // declaration (null) s = new Student(); // initialization s.name = "Alice"; // accessing field
⭐ Exam Tip: Distinguish heap (object data) vs stack (reference variable).
Easy
🔄

Life Cycle of an Object

OOP

An object's life cycle has four phases: Creation → In Use → Unreachable → Garbage Collected. Java's Garbage Collector (GC) automatically reclaims memory of unreachable objects.

  • Creation: Memory allocated, constructor called
  • In Use: Referenced by at least one variable
  • Unreachable: No active references point to it
  • GC: JVM reclaims memory; finalize() called before destruction
⭐ Exam Tip: Java does NOT have destructors like C++. Garbage Collection is automatic and non-deterministic.
Medium

Arrays in Java

Data

An Array is a fixed-size, ordered collection of elements of the same data type stored in contiguous memory. Arrays are objects in Java.

  • 1D array: int[] arr = new int[5];
  • 2D array: int[][] mat = new int[3][3];
  • Index starts at 0; last index = length-1
  • arr.length gives the size; ArrayIndexOutOfBoundsException on invalid access
  • Default values: 0 for int, null for objects, false for boolean
int[] marks = {85, 90, 78}; for(int m : marks) System.out.println(m);
⭐ Exam Tip: Arrays are zero-indexed and have a fixed size. Use ArrayList for dynamic sizing.
Easy
🔤

String in Java

Data

A String in Java is an immutable sequence of characters. Once created, its content cannot be changed. Strings are objects of the java.lang.String class.

  • Immutable: Any modification creates a new String object
  • String literals are stored in the String Pool (heap)
  • == compares references; .equals() compares content
  • Use StringBuilder / StringBuffer for mutable strings
String s1 = "Hello"; String s2 = "Hello"; s1 == s2; // true (same pool ref) s1.equals(s2); // true (same content)
⭐ Exam Tip: Always use .equals() to compare string content, never ==.
Medium
🛠️

String Methods

Data

Java's String class provides numerous built-in methods for string manipulation, comparison, and searching.

  • length() – number of characters
  • charAt(i) – char at index i
  • substring(s,e) – extract part of string
  • indexOf(str) – position of substring (-1 if not found)
  • toUpperCase() / toLowerCase() – case conversion
  • trim() – remove leading/trailing spaces
  • replace(a,b) – replace occurrences
  • split(regex) – split into array
  • contains(s) – check if substring exists
⭐ Exam Tip: Know that all these methods return a new String (immutability). Memorize the top 5 for short-answer questions.
Easy
🔗

Aggregation

OOP

Aggregation is a "HAS-A" relationship between classes where one class contains a reference to another. The contained object can exist independently.

  • Represents a weak association (independent lifecycles)
  • Example: Department HAS-A Employee — employee exists without department
  • Contrast with Composition (strong association, dependent lifecycle)
  • Enables code reusability without inheritance
class Address { String city; } class Student { Address addr; // aggregation }
⭐ Exam Tip: Aggregation = HAS-A (weak). Composition = HAS-A (strong, child dies with parent).
Medium
🏛️

Constructors

Methods

A Constructor is a special method called automatically when an object is created. It has the same name as the class and no return type.

  • Default constructor: No parameters, provided by JVM if none defined
  • Parameterized constructor: Accepts arguments for initialization
  • Copy constructor: Creates object from another object of same class
  • Cannot be static, abstract, or have a return type
  • this() calls another constructor in same class
class Box { int w, h; Box(int w, int h) { // parameterized this.w=w; this.h=h; } }
⭐ Exam Tip: If you define any constructor, JVM will NOT provide the default one automatically.
Medium
⚙️

Java Methods

Methods

A method is a block of code that performs a specific task, runs only when called, and helps with code reusability and organization.

  • Syntax: returnType methodName(params) { body }
  • void methods return nothing; others use return
  • Pass by value: Java always passes copies of arguments
  • Methods can call other methods (including themselves — recursion)
  • Instance methods need an object; static methods don't
int add(int a, int b) { return a + b; } int result = add(3, 4); // 7
⭐ Exam Tip: Java is strictly "pass by value" — even object references are passed by value (copy of the reference).
Easy
📌

Static Keyword

Modifiers

The static keyword means the member belongs to the class itself, not to any specific instance. Shared across all objects.

  • Static variable: Single copy shared by all objects (class variable)
  • Static method: Called using class name, no object needed
  • Static methods cannot access instance (non-static) members directly
  • Static block: Runs once when class is loaded
  • main() is static so JVM can call it without creating an object
class Counter { static int count = 0; // shared Counter() { count++; } }
⭐ Exam Tip: Static methods cannot use this keyword because this refers to an instance.
Medium
👆

this Keyword

OOP

this is a reference variable that refers to the current object. Used to resolve naming conflicts and to call other constructors.

  • Resolves ambiguity between instance variables and parameters with same name
  • this.field — refers to instance variable
  • this() — calls another constructor in same class (constructor chaining)
  • this can be passed as an argument to a method
  • Cannot be used inside static methods
class Person { String name; Person(String name) { this.name = name; // this = current obj } }
⭐ Exam Tip: this() must be the FIRST statement in a constructor.
Medium
🔀

Method Overloading

Methods

Overloading means defining multiple methods with the same name but different parameter lists (number, type, or order). Resolved at compile time (static polymorphism).

  • Methods must differ in: number of params, type of params, or order of params
  • Return type alone is NOT enough to overload
  • Constructors can also be overloaded
  • Also called compile-time polymorphism or early binding
int add(int a, int b) { return a+b; } double add(double a, double b) { return a+b; } int add(int a, int b, int c) { return a+b+c; }
⭐ Exam Tip: Overloading vs Overriding — Overloading is same class, different params. Overriding is subclass, same signature.
Medium
🗑️

Finalizer (finalize)

OOP

The finalize() method is called by the Garbage Collector just before destroying an object. Used to perform cleanup (e.g., close files, release resources).

  • Defined in java.lang.Object class
  • Syntax: protected void finalize() throws Throwable {}
  • Called at most once per object
  • Not guaranteed to be called — GC timing is unpredictable
  • Deprecated in Java 9+; prefer try-with-resources or AutoCloseable
⭐ Exam Tip: finalize() is NOT a destructor. You cannot force it to run. GC may never even call it.
Hard
🔒

Visibility Modifiers & Package

Modifiers

Visibility (Access) Modifiers control where a class, method, or variable can be accessed from. A Package is a namespace that groups related classes.

  • public — accessible everywhere
  • private — accessible only within the same class
  • protected — accessible within same package + subclasses
  • default (no keyword) — accessible within same package only
  • Package: declared with package com.example; at top of file
  • Import packages with import java.util.ArrayList;
package com.school; public class Student { private String name; // only here public String getName() { return name; } }
⭐ Exam Tip: Remember the access table — private < default < protected < public (increasing accessibility).
Hard
